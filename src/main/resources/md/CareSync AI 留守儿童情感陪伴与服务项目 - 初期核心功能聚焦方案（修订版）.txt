# CareSync AI 留守儿童情感陪伴与服务项目 - 初期核心功能聚焦方案（修订版）

## 一、项目核心定位

### 1. 目标愿景

聚焦留守儿童 “情感缺失” 与 “服务断层” 核心痛点，构建 “**AI 智能陪伴 + 社工精准介入**” 的轻量化服务体系。通过安全合规的 AI 交互填补日常情感空白，依托结构化数据支撑社工开展针对性服务，打造技术赋能下的留守儿童关爱新模式。

### 2. 核心价值

- **儿童端**：获得 “有温度、可信赖” 的即时情感回应，通过安全的 AI 交互满足陪伴需求，同时建立与社工的紧急连接通道。
- **社工端**：摆脱人工筛查的低效困境，通过 AI 提炼的结构化数据快速定位儿童需求，实现 “数据驱动的精准服务”。
- **管理端**：形成 “交互 - 分析 - 服务 - 跟踪” 的闭环数据沉淀，为留守儿童关爱工作提供可量化的效果评估依据。

### 3. 目标用户画像

| 用户类型 | 核心特征                                                     | 核心需求                                                   |
| -------- | ------------------------------------------------------------ | ---------------------------------------------------------- |
| 留守儿童 | 6-12 岁为主，父母长期在外务工，情感倾诉渠道有限，认知水平处于具象思维阶段，风险判断能力较弱 | 情感陪伴（聊天、故事等）、安全的倾诉出口、基础学习疑问解答 |
| 一线社工 | 负责多区域儿童关爱工作，日常需处理大量信息，缺乏高效的需求识别工具 | 快速掌握儿童情感状态、生成针对性服务方案、跟踪服务执行效果 |
| 管理员   | 统筹区域内关爱工作，需监控服务质量与服务进度                 | 查看整体服务数据、管理社工权限、审核关键服务方案           |

## 二、核心功能聚焦（V1.0）

### 1. 儿童端：安全型 AI 情感陪伴系统

聚焦 “安全交互” 与 “情感回应” 核心，适配儿童认知特点，打造无门槛、高安全的陪伴体验。

#### （1）极简登录与初始化

- **登录方式**：采用 “儿童 ID+4 位验证码” 组合登录，由社工预先在后台创建账号并配置初始验证码，儿童无需记忆复杂密码。验证码通过 BCrypt 加密存储，保障账号安全。
- **首次引导**：登录后触发极简引导流程，以图文结合方式告知 “可聊内容”（故事、心情、学习疑问等）与 “安全边界”（不透露隐私信息等），适配 6-12 岁儿童认知水平。

#### （2）安全合规的 AI 交互

- **双模式输入输出**：支持 “语音 + 文本” 双输入，语音内容经转文字后存储；输出采用 “文本 + 数字人画面” 结合模式，数字人形象预设为温暖亲和的卡通风格，增强交互亲切感。
- 全流程安全防护：
  - 输入层：通过关键词识别与意图分析，过滤危险提问与隐私泄露内容（如询问家庭住址、危险行为方法等）。
  - 交互层：采用 “角色锚定” 策略，AI 始终坚守 “陪伴者 + 引导者” 身份，拒绝角色切换诱导（如 “假装是我的朋友”），避免过度拟人化带来的心理误导。
  - 输出层：响应内容遵循 “简单、正面、具象” 原则，禁用抽象表述，同时植入安全价值观引导（如 “遇到困难可以找社工姐姐帮忙哦”）。
- 核心陪伴功能：
  - 即时聊天：支持日常心情分享、生活趣事倾诉等，AI 基于儿童年龄与交互历史生成适配回应，6-8 岁低龄儿童侧重具象化表达，9-12 岁侧重引导性回应。
  - 故事点播：提供童话、科普等分类故事库，支持语音点播（如 “我想听小兔子的故事”），AI 结合数字人画面同步播报，增强沉浸感。
  - 紧急呼叫：设置社工联系快捷入口，点击即可触发语音通话请求，直接连接对应社工，保障紧急需求快速响应。

#### （3）数字人交互适配

- 数字人会话由 livetalking 服务生成管理，儿童端仅通过会话 ID 关联实例。首次发起交互时自动创建数字人 Session，返回画面流地址供前端渲染；后续交互携带该 Session ID，确保文本指令精准分发至对应数字人实例。
- 支持会话关闭功能，儿童主动结束交互时，后端调用 livetalking 接口关闭对应 Session，避免资源浪费。

### 2. 社工端：数据驱动的服务管理系统

以 “AI 分析结果” 为核心入口，构建 “儿童管理 - 需求识别 - 方案执行 - 进度跟踪” 的全流程服务工具。

#### （1）儿童信息管理

- 支持 “批量创建 + 单个编辑” 儿童账号，录入姓名、年龄、监护人电话等基础信息，可上传儿童兴趣偏好标签（如 “喜欢绘画”“爱听童话”），为 AI 交互提供个性化依据。
- 首页展示 “有新消息” 儿童列表（基于`child`表`has_new_chat`字段筛选），标注未读消息数量与最新情感标签，优先处理高需求儿童。

#### （2）AI 分析结果查看

- 每个儿童详情页展示 AI 提炼的结构化信息：情感趋势（如 “近 3 天：孤独→平静→开心”）、核心需求（如 “故事陪伴、数学疑问”）、潜在问题（如 “多次提到想妈妈”），数据来源于`child`表`ai_struct_info`字段。
- 提供 “分析溯源” 功能，点击可查看生成该结果的原始聊天记录，每条记录标注情感标签与过滤状态，辅助社工验证需求真实性。

#### （3）服务方案生成与管理

- **AI 建议生成**：基于儿童 30 天内的交互数据，AI 自动生成服务建议，包括目标建议（如 “缓解孤独感”）、措施建议（如 “每周 3 次数字人故事推送”）及依据说明（如 “近 5 次对话 3 次提到想听故事”）。
- **方案编辑与审核**：社工可基于 AI 建议调整方案，明确服务目标、具体措施（最多 3 条，确保可执行性）、服务周期（默认 7 天），管理员账号需对 “高风险问题”（如心理疏导类）方案进行审核。
- 方案状态分为 “草稿 - 进行中 - 已完成” 三类，状态变更同步更新至数据库，支持按状态筛选查询。

#### （4）服务进度跟踪

- 按服务周次记录执行情况，每次可录入 “完成状态”（已完成 / 未完成 / 延迟）、完成率及具体内容（如 “已完成 1 次电话沟通，儿童反馈开心”）。
- 支持上传照片、录音等佐证材料，形成完整的服务档案，便于后续追溯与效果评估。

#### （5）账号与权限管理（管理员专属）

- 管理社工账号，配置 “普通社工 / 管理员” 角色权限，管理员可重置社工密码、禁用账号，保障系统操作安全。
- 查看全区域服务数据统计，包括 “活跃儿童数”“方案完成率”“高频需求类型” 等指标，生成可视化报表。

### 3. 系统后台：自动化数据处理与安全保障

#### （1）AI 分析引擎

- 采用 “定时 + 触发” 双模式分析：每日凌晨自动分析所有儿童近 24 小时聊天记录，同时支持社工手动触发特定儿童的深度分析。
- 分析结果同步更新至`child`表`ai_struct_info`字段与`ai_analysis_log`表，记录分析涉及的会话范围与触发方式（自动 / 手动）。

#### （2）内容安全过滤

- 聊天内容实时经过双重过滤：先通过关键词匹配拦截明显违规内容，再通过 AI 意图识别判断潜在风险（如模糊询问危险行为），过滤状态标记于`ai_chat_record`表`is_filtered`字段。
- 对涉及 “自我伤害、家庭冲突” 等高危内容的交互，即时标记并推送提醒给对应社工。

#### （3）数据存储与索引优化

- 核心业务数据按表结构化存储，动态信息（如 AI 分析结果、兴趣标签）采用 JSONB 类型存储，兼顾查询效率与扩展灵活性。
- 针对高频查询场景创建索引：儿童登录 ID 索引、新消息筛选索引、会话查询索引等，保障系统响应速度。

## 三、技术实现核心设计

### 1. 技术栈选型

| 技术层面   | 选型方案              | 选型依据                                                     |
| ---------- | --------------------- | ------------------------------------------------------------ |
| 后端框架   | SpringBoot 3.x        | 轻量化开发，支持快速集成第三方服务，适配小团队敏捷迭代需求   |
| 数据库     | PostgreSQL 15.x       | 原生支持 JSONB 类型与复杂查询，满足结构化与非结构化数据存储需求 |
| 前端框架   | Vue 3 + Element Plus  | 组件化开发效率高，UI 组件适配管理系统场景，学习成本低        |
| AI 模型    | 定制化儿童专用大模型  | 优化儿童语言理解能力，内置安全规则，避免通用模型的风险隐患   |
| 语音处理   | 阿里云 ASR/TTS        | 识别准确率高，支持儿童语音优化，提供稳定的 API 服务          |
| 数字人交互 | 对接 livetalking 服务 | 专注数字人实例管理与画面传输，后端仅维护会话关联，降低开发复杂度 |

### 2. 核心数据流程

#### （1）AI 交互数据流程

1. 儿童发起交互（语音 / 文本），前端携带业务会话 ID 发送至后端；
2. 后端若未获取到数字人 Session ID，调用 livetalking 接口创建 Session，获取`digi_session_id`与画面流地址；
3. 语音内容转文字后进行安全过滤，通过则调用 AI 模型生成回复，同步发送文本指令至 livetalking 服务更新数字人画面；
4. 儿童消息与 AI 回复均存储至`ai_chat_record`表，关联`child_id`与`digi_session_id`；
5. 更新`child`表`has_new_chat`字段为`TRUE`，触发社工端提醒。

#### （2）服务方案数据流程

1. AI 定时分析`ai_chat_record`表数据，生成结构化结果存储至`ai_analysis_log`表；
2. 社工查看分析结果，基于`ai_analysis_id`关联生成服务方案，存储至`ai_assist_scheme`表；
3. 社工执行服务后，录入跟踪日志至`assist_track_log`表，关联`scheme_id`；
4. 方案完成后，系统自动汇总周次完成率，更新方案状态为 “已完成”。

### 3. 关键技术保障

- **安全防护**：实现数据传输 HTTPS 加密、敏感信息（密码、验证码）BCrypt 加密存储，数字人交互采用 “最小权限原则”，禁用无关功能。
- **性能保障**：通过索引优化与查询缓存，确保单表百万级数据下查询响应时间≤300ms，支持同时在线用户≥500 人。
- **可靠性保障**：关键操作（方案创建、日志录入）实现事务控制，聊天记录采用本地备份 + 定期归档策略，避免数据丢失。

## 四、产品交互与体验设计

### 1. 儿童端交互设计

- **界面风格**：采用明亮柔和的色彩体系，图标为卡通化设计，避免复杂文字描述，核心功能入口（聊天、故事、呼叫）占据屏幕主要区域。
- **交互逻辑**：遵循 “一次操作完成一个任务” 原则，语音交互支持打断重说，故事播放提供 “暂停 / 重播” 极简控制，适配儿童操作习惯。
- **反馈机制**：数字人通过动态表情（微笑、点头）与语音语调变化回应情绪，文本回复搭配对应表情包，增强情感连接感。

### 2. 社工端交互设计

- **信息层级**：首页突出 “高优先级儿童”（新消息、高危标签），详情页采用 “分析结果 - 原始记录 - 方案管理” 的递进式布局，核心数据前置展示。
- **操作效率**：方案生成提供 “AI 建议一键导入” 功能，跟踪日志支持 “模板快速填充”，减少重复输入工作。
- **视觉提示**：采用颜色编码区分紧急程度（红色 = 高危需求，黄色 = 待处理方案，绿色 = 正常状态），辅助社工快速识别重点。

## 五、实施与落地规划

### 1. 开发阶段（4 周）

| 阶段           | 时间      | 核心任务                                                     | 交付物                                   |
| -------------- | --------- | ------------------------------------------------------------ | ---------------------------------------- |
| 需求细化与设计 | 第 1 周   | 确定 AI 交互话术规则、绘制 UI 原型、设计数据库表结构         | 需求规格说明书、UI 原型稿、建表 SQL 脚本 |
| 核心功能开发   | 第 2-3 周 | 儿童端登录与交互功能、社工端管理与服务功能、AI 与 livetalking 对接 | 可运行的前后端代码、接口文档             |
| 测试与优化     | 第 4 周   | 功能测试（覆盖 80% 核心场景）、安全测试（内容过滤 / 数据加密）、性能测试 | 测试报告、优化后的系统版本               |

### 2. 试点落地（2 周）

- **试点范围**：选取 1 个乡镇作为试点，覆盖 50 名留守儿童与 5 名社工。
- 落地步骤：
  1. 系统部署与账号配置：完成服务器部署，为试点儿童创建账号，对社工进行操作培训；
  2. 试运行与问题收集：上线后安排 1 周试运行，每日收集儿童使用反馈与社工操作问题；
  3. 迭代优化：针对试点问题进行快速迭代，优化交互体验与功能稳定性。

### 3. 上线推广（后续阶段）

- 基于试点数据优化系统后，按 “乡镇 - 区县 - 地市” 梯度推广，同步完善内容库（故事、科普等）。
- 建立 “月度迭代” 机制，根据用户反馈新增功能（如分龄教育内容、家长端查看权限）。

## 六、风险与应对策略

| 风险类型 | 具体表现                                      | 应对策略                                                     |
| -------- | --------------------------------------------- | ------------------------------------------------------------ |
| 安全风险 | 儿童泄露隐私信息、AI 生成不当内容、账号被盗用 | 1. 输入层实时拦截隐私关键词，输出层强化价值观引导；2. 定期更新安全过滤规则库；3. 验证码定期强制重置，限制单 IP 登录次数 |
| 体验风险 | 儿童对 AI 交互不感兴趣、社工认为数据不准确    | 1. 前期调研儿童兴趣，丰富故事 / 科普内容；2. 优化 AI 模型，提升情感标签与需求识别准确率；3. 保留人工修正 AI 分析结果的入口 |
| 技术风险 | 数字人会话连接失败、AI 分析响应缓慢           | 1. 实现数字人 Session 自动重连机制，失败时降级为纯文本交互；2. 优化 AI 分析算法，采用增量分析替代全量分析；3. 配置服务监控，异常时触发告警 |
| 运营风险 | 社工使用意愿低、儿童活跃度下降                | 1. 设计简洁操作手册，开展 “一对一” 培训；2. 建立社工激励机制（如服务效果排行榜）；3. 定期更新内容库，保持交互新鲜感 |

## 七、核心成功指标

### 1. 儿童端指标

- 日均活跃率≥40%（每周至少使用 3 天）；
- 单次交互时长≥3 分钟；
- 安全过滤触发率≤5%（反映交互内容合规性）；
- 紧急呼叫功能月使用率≥2%（体现功能实用性）。

### 2. 社工端指标

- 服务方案生成周期≤1 天（从查看分析结果到方案创建完成）；
- 方案完成率≥80%；
- 社工对 AI 分析结果的满意度≥85%（通过问卷调查）。

### 3. 系统指标

- 服务可用性≥99.9%；
- 核心接口响应时间≤500ms；
- 数据存储安全性 100%（无泄露 / 丢失事件）。

## 八、方案亮点

1. **轻量高效**：移除独立数字人会话表，将关联逻辑嵌入聊天记录表，降低系统复杂度，适配小团队快速开发与落地需求。
2. **安全优先**：基于儿童认知特点构建 “输入 - 交互 - 输出” 全流程安全防护，符合《未成年人网络保护条例》要求，杜绝不良信息渗透风险。
3. **数据闭环**：从 AI 交互数据到服务跟踪日志形成完整闭环，既为社工提供决策依据，也为项目效果评估提供可量化支撑。
4. **体验适配**：儿童端极简交互适配低操作能力，社工端数据前置适配高效工作需求，实现 “不同角色、同样友好” 的体验设计。

# CareSync AI 项目技术文档

## 一、项目概述

### 1.1 项目背景

随着我国城镇化进程加速，留守儿童问题已成为社会关注的焦点。据统计，我国农村留守儿童数量超过6900万，这些儿童长期缺乏父母的陪伴和关爱，面临情感需求难以满足、心理健康问题频发等挑战。传统的社工服务模式存在覆盖面有限、专业资源不足、服务效率低下等问题，难以满足大规模留守儿童的情感陪伴和精准服务需求。

在此背景下，CareSync AI项目应运而生，旨在通过人工智能技术与社会工作服务深度融合，构建"AI智能陪伴+社工精准介入"的轻量化服务体系，为留守儿童提供有温度的情感陪伴，并为社工提供数据驱动的精准服务支持。

### 1.2 项目定位

CareSync AI项目聚焦留守儿童情感陪伴与服务领域，致力于打造一个连接儿童、社工和管理方的智能化服务平台。项目以AI技术为核心驱动力，通过安全合规的AI交互填补日常情感空白，依托结构化数据支撑社工开展针对性服务，最终形成交互-分析-服务-跟踪的闭环数据沉淀与效果评估体系。

### 1.3 核心价值

- **儿童端**：提供有温度、可信赖的即时情感回应和安全的倾诉出口，有效缓解留守儿童情感孤独问题
- **社工端**：通过AI提炼的结构化数据快速定位儿童需求，实现数据驱动的精准服务，提升服务效率和质量
- **管理端**：形成完整的服务闭环数据，提供可量化的效果评估依据，为政策制定和资源配置提供数据支持

## 二、技术架构与实现方案

### 2.1 整体架构设计

CareSync AI项目采用分层架构设计，包括前端展示层、后端服务层、数据存储层和AI能力层四个核心层次，实现了业务逻辑与数据存储的分离，AI能力与业务应用的解耦，为系统的可扩展性和稳定性提供了保障。

![系统架构图](https://placeholder-for-architecture-diagram)（注：实际文档中请替换为真实架构图）

**核心架构说明：**

1. **前端展示层**：采用现代化HTML+CSS+JavaScript技术栈，结合Tailwind CSS和Font Awesome等UI组件库，构建响应式、易用的用户界面
2. **后端服务层**：基于Spring Boot 3.x框架，实现业务逻辑处理、API接口提供和权限管理等功能
3. **数据存储层**：使用PostgreSQL 15.x数据库，支持JSONB类型存储非结构化数据，结合MyBatis-Plus实现高效的数据访问
4. **AI能力层**：集成定制化儿童专用大模型、语音识别/合成服务和数字人交互能力，为系统提供核心AI支持

### 2.3 核心功能模块实现

#### 2.3.1 儿童管理模块

儿童管理模块是系统的基础模块，负责儿童信息的录入、查询、更新和删除等功能，支持社工对服务对象进行全面管理。

**核心功能点：**
- 儿童信息的增删改查，支持多条件搜索和筛选
- 服务状态和风险等级管理，实现动态评估
- 验证码生成与管理，保障儿童登录安全

**技术实现：**
```java
@RestController
@RequestMapping("/api/social-worker/children")
@Tag(name = "儿童管理模块接口", description = "社工管理儿童相关接口")
public class ChildManageController {
    // 儿童列表分页查询
    @PostMapping("/list")
    @Operation(summary = "获取儿童列表", description = "分页查询儿童列表")
    public Result<PageResult<ChildQueueVO>> getChildList(@RequestBody ChildQueryDTO childQueryDTO) {
        PageResult<ChildQueueVO> result = childService.getChildList(childQueryDTO);
        return Result.success(result);
    }
    // 其他接口实现...
}
```

#### 2.3.2 AI分析模块

AI分析模块是系统的核心能力模块，负责对儿童与AI的交互会话进行智能分析，提取关键信息和情感倾向，为社工提供决策支持。

**核心功能点：**
- 自动分析儿童与AI的会话记录，识别潜在问题
- 生成结构化的分析报告，包括情感趋势、潜在问题等
- 支持多维度筛选和查询分析结果

**技术实现：**
系统采用PostgreSQL的JSONB类型存储非结构化的分析结果，结合MyBatis-Plus的TypeHandler机制实现对象与JSON数据的自动转换，提高开发效率和系统性能。

```java
@Getter
@Setter
@TableName("ai_analysis_log")
public class AiAnalysisLog extends Model<AiAnalysisLog> {
    @TableField(value = "analysis_result", typeHandler = JsonbTypeHandler.class)
    private Object analysisResult;
    // 其他字段...
}
```

#### 2.3.3 服务方案模块

服务方案模块是连接AI分析与社工服务的桥梁，支持AI根据分析结果生成服务方案，并允许社工进行调整和执行。

**核心功能点：**
- AI一键生成服务方案，包括目标、措施、周期等
- 支持社工对方案进行编辑和调整
- 方案状态管理，实现全生命周期跟踪

**技术实现：**
```java
@PostMapping("/generate")
@Operation(summary = "生成AI服务方案", description = "一键生成AI服务方案")
public Result<AssistSchemeVO> generateScheme(@RequestBody GenerateSchemeDTO generateSchemeDTO) {
    try {
        AssistSchemeVO result = aiAssistSchemeService.generateScheme(generateSchemeDTO);
        return Result.success(result);
    } catch (Exception e) {
        log.error("生成服务方案失败", e);
        return Result.error("生成服务方案失败，请稍后重试");
    }
}
```

#### 2.3.4 进度跟踪模块

进度跟踪模块负责记录服务方案的执行进度和效果，支持社工定期更新服务进展，并生成可视化的进度报告。

**核心功能点：**
- 记录服务方案的执行进度和完成情况
- 支持按多种维度查询和筛选进度日志
- 生成可视化的进度报告，支持导出

**技术实现：**
系统设计了完善的进度日志记录机制，包括按时间倒序的分页查询、按方案ID和儿童ID的关联查询等，方便社工随时了解服务进展。

## 三、人工智能技术应用

### 3.1 定制化儿童专用大模型

CareSync AI项目采用定制化的儿童专用大模型，针对儿童语言特点和心理需求进行了优化，具有以下特点：

1. **儿童语言理解优化**：针对儿童口语表达特点进行训练，提高对儿童语言的理解准确性
2. **内置安全规则**：严格过滤不适合儿童的内容，确保交互的安全性和健康性
3. **情感陪伴能力**：具备共情能力，能够给予儿童适当的情感回应和心理支持
4. **问题识别能力**：能够识别儿童表达中潜在的心理问题和风险信号

### 3.2 会话内容智能分析

系统通过AI技术对儿童与AI的会话内容进行智能分析，实现以下功能：

1. **情感分析**：识别儿童的情绪状态和情感倾向，如开心、焦虑、孤独等
2. **主题提取**：提取会话中的核心主题和关注点
3. **风险识别**：识别潜在的心理问题和风险信号，如抑郁倾向、自残倾向等
4. **趋势分析**：分析儿童情绪和问题的变化趋势，为干预提供依据

### 3.3 AI辅助服务方案生成

基于对儿童情况的分析，AI能够自动生成个性化的服务方案，包括：

1. **服务目标设定**：根据儿童的具体情况设定合理的服务目标
2. **干预措施建议**：提供具体的干预方法和活动建议
3. **服务周期规划**：根据问题的严重程度和复杂性规划服务周期
4. **评估指标制定**：设定服务效果的评估指标和方法

### 3.4 语音识别与合成技术

项目集成了阿里云ASR/TTS服务，实现语音交互功能：

1. **儿童语音识别**：针对儿童语音特点进行优化，提高识别准确率
2. **情感语音合成**：合成带有适当情感色彩的语音，增强交互的自然性和亲切感
3. **多语言支持**：支持多种方言和语言，满足不同地区儿童的需求

## 四、创新点与技术优势

### 4.1 模式创新：AI+社工协同服务模式

CareSync AI项目创新性地提出了"AI智能陪伴+社工精准介入"的服务模式，通过AI技术弥补社工资源不足的问题，同时通过社工的专业干预确保服务质量，实现了技术与人文关怀的有机结合。

### 4.2 技术创新：定制化儿童AI模型

项目开发了定制化的儿童专用大模型，针对儿童语言特点和心理需求进行了深度优化，解决了通用AI模型在儿童服务场景中的适应性问题，提高了交互的准确性和安全性。

### 4.3 数据创新：结构化数据支撑精准服务

系统通过AI技术将非结构化的会话内容转化为结构化的分析结果，为社工提供清晰、直观的数据支持，帮助社工快速定位问题和制定干预策略，大大提高了服务的精准性和效率。

### 4.4 体验创新：多模态交互方式

项目支持文字、语音、数字人等多种交互方式，根据儿童的偏好和能力提供个性化的交互体验，增强了儿童的参与感和舒适度。
